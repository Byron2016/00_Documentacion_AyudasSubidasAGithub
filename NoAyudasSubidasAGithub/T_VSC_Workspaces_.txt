			- Workspaces vs Profiles 
				- Workspaces
					- Son espacios de trabajo que se enfocan en la gestión y configuración de proyectos.
					- Puedes abrir el workspaces y el editor te muestre todos los directorios que pertenecen a ese workspaces aún cuando estén en diferentes lugares. 
					- Maneja lo que son extensiones y recomendaciones específicas de extensiones para el proyecto.
					- Puede ser compartido con el equipo de desarrollo 
					- 
					
				- Profiles 
					- Se centran más en experiencia de desarrollo, temas como aspecto de VSC, comandos específicos, combinaciones de teclas, extensiones instaladas para cada perfil. 
					- Para un mismo workspace puedo tener diferentes perfiles, lo que podrá permitierme cambiar de uno a otro, por ejemplo perfil-desarrollo y perfil-testing.

	- Configuring recommended extensions
		- https://dev.to/askrishnapravin/recommend-vs-code-extensions-to-your-future-teammates-4gkb
			- CMD + Shift + p
			- Type Configure Recommended Extensions
			- A new file called extensions.json will be created under .vscode/ in the root of the project.
			- Place the cursor within the recommendations array and press CMD + i
			- Select the required extensions from the list of installed extensions and save the file.
			- Include this file in your source control tools(we use Git).
